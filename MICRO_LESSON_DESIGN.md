# CodeQuest Micro-Lesson Design

Every lesson should follow this order:

1. **Core idea** — the smallest accurate explanation of the concept.
2. **Mental model** — a memorable way to reason about it.
3. **Worked example** — one concrete use.
4. **Common mistake** — a misconception or failure mode.
5. **Pause and predict** — retrieval before execution.
6. **Guided build** — the existing code, query or interface task.
7. **Explanation** — the learner should be able to describe why the solution works.

Theory should normally fit within 60–90 seconds of reading. It should not become a textbook
chapter or merely restate syntax.
